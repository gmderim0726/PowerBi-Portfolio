# Credit Risk Analysis & Operational Monitoring

## The problem

A bank needed a clean visualization of credit usage to mitigate default risk and reduce costumer churn. The project's goal was to build an ent-to-end pipeline to ingest raw data and create management and operational dashboards.

## Source Data

The dataset used in this project was obtained from **Kaggle**. It contains information about credit card clients, including their demographic profiles, credit limits, and transaction behaviors. 
* **Link to the dataset:** [https://www.kaggle.com/datasets/sakshigoyal7/credit-card-customers]

## Key questions explored

1. What are the total values for credit limit, revolving balance, and churn rate?
2. Who has the highest credit limit?
3. Is the churn rate under control?
4. Does the card category affect usage?
5. Who is closest do default risk?
6. Who is the profitable target audience?
7. Are the customers satisfied?

## Getting started

To get start with Credit Analysis, you can follow these steps:

1. Clone this repository to your local machine.
2. In the Pyton pipeline script, insert your PostgreSQL server information.
3. Run the script to load the dataset into PostgreSQL.
4. Execute the provided SQL queries in PostgreSQL to create the views.
5. Open your Power BI file to interact with visualizations and answer the Key questions.
